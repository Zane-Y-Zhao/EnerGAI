# 化工过程智能决策系统演示案例

## 案例设计原则
- **故事性**：每个案例包含"问题发生→系统响应→决策依据→业务价值"四幕
- **真实性**：全部基于杨泽彤提供的TE数据集与故障案例
- **协作性**：每个案例必须显性标注三方贡献

## 核心演示案例

### Case-1 温度骤升预警

**场景**：温度骤升预警

**输入数据（冯申雨）**：
```json
{
  "temperature": 85.5,
  "pressure": 4.2,
  "flow_rate": 10.5,
  "heat_value": 1250.8,
  "timestamp": "2026-04-12T14:30:00",
  "unit": "°C"
}
```

**系统输出（韩永盛）**：
```
"立即关闭FV-101阀门（依据：杨泽彤-系统操作规则文档_v2.pdf第3.2条）"
```

**三方溯源（答辩话术）**：
"这是韩永盛模型发现的温度异常，冯申雨知识库精准定位处置规则，杨泽彤签署的安全条款确保操作合法"

**业务价值**：及时预警温度异常，避免设备损坏和安全事故

### Case-3 单位校验拦截

**场景**：单位校验拦截

**输入数据（冯申雨）**：
```json
{
  "temperature": 300,
  "pressure": 3.0,
  "flow_rate": 10.0,
  "heat_value": 1000.0,
  "timestamp": "2026-04-12T14:40:00",
  "unit": "K"
}
```

**系统输出（韩永盛）**：
```
"[ERROR] 物理量单位 'K' 未在杨泽彤《参数配置表》中注册"
```

**三方溯源（答辩话术）**：
"单位是连接物理世界与数字模型的桥梁，此错误拦截证明系统对杨泽彤权威参数的绝对尊重"

**业务价值**：防止因单位错误导致的决策失误，确保系统操作的准确性

### Case-4 多轮对话跟踪

**场景**：多轮对话跟踪

**第1轮输入（冯申雨）**：
```json
{
  "temperature": 85.5,
  "pressure": 4.2,
  "flow_rate": 10.5,
  "heat_value": 1250.8,
  "timestamp": "2026-04-12T14:45:00",
  "unit": "°C"
}
```

**第1轮输出（韩永盛）**：
```
"立即关闭FV-101阀门（依据：杨泽彤-系统操作规则文档_v2.pdf第3.2条）"
```

**第2轮输入（赵元卿）**：
```json
{
  "session_id": "session_123",
  "message": "阀门状态如何？"
}
```

**第2轮输出（韩永盛）**：

**业务价值**：支持多轮对话，提高用户体验和决策连贯性

### Case-5 安全熔断演示

**场景**：安全熔断演示

**操作**：删除安全规程文件后发送请求

**输入数据（冯申雨）**：
```json
{
  "temperature": 85.5,
  "pressure": 4.2,
  "flow_rate": 10.5,
  "heat_value": 1250.8,
  "timestamp": "2026-04-12T14:50:00",
  "unit": "°C"
}
```

**系统输出（韩永盛）**：
```
"[ERROR] 安全条款缺失，请人工介入"
```

**三方溯源（答辩话术）**：
"没有杨泽彤的安全条款，系统宁可停摆也不越界——这是化工AI的伦理底线"

**业务价值**：确保系统在安全条款缺失时不会做出可能危险的决策，保障生产安全
