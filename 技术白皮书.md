## 2. 核心模块实现细节

### 2.1 知识库构建

#### 2.1.1 多源文档清洗

`clean_text()` 实现了工业文档专用清洗，保留专业术语，剔除干扰符号：

- 移除多余空白与换行：`re.sub(r'\s+', ' ', text)`
- 清除页眉页脚：`re.sub(r'第\s*\d+\s*页|共\s*\d+\s*页|©\d{4}.*', '', text)`
- 过滤孤立编号：`re.sub(r'^\d+\.\s*$', '', text, flags=re.MULTILINE)`
- 保留中英文、数字、常用标点和工程符号（如Δ、MPa、°C、kg/s等）
- 去除连续无意义字符：`re.sub(r'(.)\1{2,}', r'\1', text)`

#### 2.1.2 语义分块策略

`split_into_chunks()` 采用 `chunk_size=300`、`overlap=50` 策略：

- 优先在句号、分号、换行处切分，避免截断化工术语
- 带单位数值（如"温度=85.5°C"）在分块时保持完整
- 50字重叠确保语义连续性，防止关键信息孤岛

### 2.2 RAG检索增强（冯申雨）

#### 2.2.1 HyDE技术

`generate_hypothetical_answer()` 利用通义千问先生成假设性答案，再以此进行向量检索，提升对未明确表述问题的召回率：

```python
def hyde_retriever(query):
    # 1. 生成假设答案
    hypothetical = call_qwen(f"作为工业能源专家，请基于问题给出详细分析：{query}")
    # 2. 用假设答案检索
    docs = vectorstore.similarity_search(hypothetical, k=3)
    return docs
```

#### 2.2.2 安全条款硬过滤

`get_safety_rules()` 强制从向量库中检索"安全操作规程"类文档，确保每一条诊断建议均含安全边界约束：

- 过滤条件：`filter={"source": {"$contains": "安全操作规程"}}`
- 与ISA-88/GB标准对齐，杜绝不安全建议生成

### 2.3 Transformer预测引擎（韩永盛）

#### 2.3.1 多变量时序预测

模型输入为多变量历史序列（温度、压力、流量、能耗），输出未来5-15分钟的预测趋势及置信区间。

#### 2.3.2 AutoResearch自动调参

AutoResearch框架自动监测训练过程，动态调整：

- Dropout率（防止过拟合）
- 学习率衰减策略
- 注意力头数与网络层数

实现模型在非平稳工业数据上的鲁棒预测。

#### 2.3.3 注意力机制可解释性

模型输出注意力权重热力图，标识对预测贡献最大的时间点和变量。例如："'蒸汽压力'在20分钟前的突变是本次效率下降的主要原因"，为操作员提供直观的解释。

### 2.4 决策流水线（赵元卿集成）

#### 2.4.1 双引擎协同流程

```
传感器数据 → Transformer预测 (韩) → 决策API (赵/冯)
    → RAG检索 (冯) + 安全条款注入 → 通义千问生成决策建议 (冯)
        → SSE推送到前端 (赵) → 智能问答界面展示
```

#### 2.4.2 溯源机制

每份决策建议包含 `source_trace` 字段，记录：

- **预测来源**：Transformer模型版本、输入数据时间戳
- **知识来源**：检索到的文档名称及分块ID
- **安全依据**：引用的国家标准/操作规程编号（如GB/T 50050-2017）

## 3. 工业级保障机制

### 3.1 错误码体系

| 错误码 | 含义 | 描述 |
|--------|------|------|
| 200 | OK | 请求成功 |
| 422 | UNIT_MISMATCH | 单位不匹配（仅支持SI标准单位） |
| 400 | BAD_REQUEST | 参数缺失或格式错误 |
| 500 | INTERNAL_ERROR | 模型推理失败或服务器内部异常 |
| 503 | MODEL_TIMEOUT | 模型计算超时，自动降级处理 |

### 3.2 日志审计

- 所有API请求、AI决策、异常事件记录到 `logs/decision_api.log`
- 格式：`时间戳 | 级别 | 模块 | 消息`
- 满足72小时滚动留存，便于复盘与安全审计
