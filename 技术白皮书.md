## 2. 核心模块实现细节

### 2.1 知识库构建（Day 2）

#### 2.1.1 清洗逻辑

`clean_text()` 函数（document_processor.py:11-23）实现了化工文档专用清洗，保留专业术语，剔除无意义符号：

- 移除多余空白与换行：`re.sub(r'\s+', ' ', text)`
- 移除页眉页脚常见模式：`re.sub(r'第\s*\d+\s*页\s*共\s*\d+\s*页|©\d{4}.*', '', text)`
- 移除孤立数字编号：`re.sub(r'^\d+\.\s*$', '', text, flags=re.MULTILINE)`
- 只保留中文、英文、数字、常用标点和特殊符号（如Δ、=、MPa等）：`re.sub(r'[^\u4e00-\u9fa5a-zA-Z0-9\s\u002E\u002D\u002C\u003B\u0021\u003F\u0028\u0029\u003D\u0394]', '', text)`
- 移除连续的无意义字符：`re.sub(r'(.)\1{2,}', r'\1', text)`

#### 2.1.2 分块策略

`split_into_chunks()` 函数（document_processor.py:113-158）实现了按语义切分，优先在句号/分号/换行处分割，避免切断化工术语和带单位数值：

- 定义带单位数值的模式：`unit_pattern = r'[A-Za-zΔ]+\s*=\s*\d+(\.\d+)?[A-Za-z°]+|\d+(\.\d+)?[A-Za-z°]+'`
- 先标记带单位数值，避免被分割
- 按句号/分号/换行处分割：`sentences = re.split(r'(?<=[。；！？])\s+|[\r\n]+', text)`
- 确保重叠部分不会切断带单位数值

**为什么使用 `chunk_size=300` 且 `overlap=50`：**
- 保障阀门条件等关键信息不被切碎
- 300字的块大小适合向量嵌入和语义理解
- 50字的重叠确保上下文连贯性，避免重要信息丢失

### 2.2 RAG检索增强（Day 2–5）

#### 2.2.1 HyDE技术原理

`generate_hypothetical_answer()` 函数（rag_pipeline.py:93-111）实现了HyDE技术，用千问生成假设答案提升召回率：

```python
# 伪代码
def hyde_retriever(query):
    # 1. 生成假设答案
    hypothetical_answer = call_qwen(f"你是化工专家，基于问题生成详细答案：{query}")
    # 2. 使用假设答案进行检索
    results = vectorstore.similarity_search(hypothetical_answer, k=3)
    # 3. 返回检索结果
    return results
```

#### 2.2.2 安全条款硬过滤

`get_safety_rules()` 函数（prompt_engineering.py:87-94）实现了安全条款硬过滤，确保决策过程中必须包含安全操作规程：

```python
def get_safety_rules(vectorstore, top_k=1):
    """从Chroma中精准检索安全类规则"""
    results = vectorstore.similarity_search(
        query="高温管道表面温度限值、泄漏应急处置、安全操作边界",
        k=top_k,
        filter={"source": {"$contains": "安全操作规程"}}
    )
    return results
```

**强制约束意义：**
- 确保所有决策建议都基于安全操作规程
- 防止不安全的操作建议被生成
- 符合化工行业的安全规范要求

### 2.3 决策流水线（Day 3–4）
