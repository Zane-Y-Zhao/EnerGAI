# 系统使用手册

## 1. 系统简介
- 系统名称：化工余热智能管理系统（Chemical-Heat-Recovery-System）
- 版本号：v1.0.0（决策 API 标注版本）
- 适用对象：化工工艺工程师、值班操作员、运维与测试人员
- 主要功能概述：
  - 监控余热温度、系统压力、余热回收、节能效果等关键指标
  - 展示趋势分析、设备状态、预警信息
  - 提供智能操作建议，并支持查看决策溯源信息
  - 提供智能助手会话能力（支持会话上下文）

## 2. 运行环境要求
- 操作系统：Windows 10/11、Linux、macOS（开发环境）
- 运行时：Python 3.10+（推荐）
- 浏览器/客户端：Chrome、Edge（推荐最新版本）
- 硬件要求：
  - 最低：4 核 CPU，8 GB 内存
  - 推荐：8 核 CPU，16 GB 内存（启用知识库检索和模型时）
- 网络要求：
  - 本地开发可离线运行前端模拟数据
  - 若启用外部模型服务，需可访问对应模型接口

## 3. 安装与部署
### 3.1 前置准备
- 获取项目源码并进入项目根目录
- 准备 Python 环境并安装依赖（项目未提供统一 requirements，按实际模块安装 FastAPI、Uvicorn、Pydantic、NumPy、LangChain、Chroma、rank-bm25、HuggingFace Embeddings 等）
- 确保端口可用：
  - 8000：后端+前端静态资源服务（`backend/main.py`）
  - 8001：智能决策 API（`start_api.py`）
  - 8006：`api/main.py` 直接启动时的默认端口（可选）

### 3.2 安装步骤
1. 打开终端进入项目根目录。
2. 创建并激活虚拟环境（可选但推荐）。
3. 安装依赖并确认 Python 可运行 FastAPI 服务。
4. 启动智能决策 API（见 3.3）。
5. 启动后端服务（见 3.3）。
6. 浏览器访问后端地址查看系统大屏。

### 3.3 启动与停止
- 启动命令（推荐）：
  - 启动决策服务（端口 8001）：
    - `python start_api.py`
  - 启动后端服务（自动寻找 8000 起可用端口）：
    - `python backend/main.py`
  - 启动后访问：
    - `http://127.0.0.1:8000/`（若 8000 被占用，请按控制台提示端口访问）
- 可选启动命令：
  - 直接启动 `api/main.py`（默认 8006）：
    - `python api/main.py`
- 停止命令：
  - 在对应终端按 `Ctrl + C` 停止服务

## 4. 功能使用说明
### 4.1 登录系统
- 当前版本未实现用户名/密码登录。
- 启动服务后，直接在浏览器访问首页即可进入系统主界面。

### 4.2 主界面说明
- 页面名称：化工余热智能管理系统大屏
- 主要区域：
  - 综合概览（KPI 卡片）
  - 趋势分析（温度/压力/回收热量）
  - 能耗分析（柱状图）
  - 设备状态（健康度与状态）
  - 预警信息
  - 智能操作建议（支持查看溯源、执行、忽略）
  - 智能助手会话区（支持消息交互与依据展示）
- 数据更新频率：
  - 时间每 1 秒刷新
  - 业务数据每 5 秒刷新一次

### 4.3 核心功能操作
#### 4.3.1 监控与可视化
- 查看实时 KPI（温度、压力、余热回收、节能效果）及预测值
- 查看趋势分析图与能耗分析图
- 点击“导出CSV”下载当前数据
- 点击“导出图表”下载趋势图/能耗图 PNG 图片

#### 4.3.2 智能决策建议
- 系统自动加载决策建议
- 可执行操作：
  - “立即执行”：记录执行动作（当前为前端占位逻辑）
  - “忽略”：记录忽略动作（当前为前端占位逻辑）
  - “查看依据”：展示 `source_trace` 溯源信息
- 决策接口（推荐）：`POST /api/v1/decision`

#### 4.3.3 智能助手会话
- 在“智能助手”输入问题后点击“发送”或按回车
- 系统返回助手回复并展示“操作依据（context_trace）”
- 会话接口：`POST /api/v1/conversation`
- 支持基于 `session_id` 的上下文继承（例如关联历史阀门编号）

## 5. 常见问题与排查
- 问题现象：页面无法打开
  - 可能原因：后端服务未启动或端口冲突
  - 解决方法：重新执行 `python backend/main.py`，按控制台提示端口访问
- 问题现象：智能助手无响应或报错
  - 可能原因：决策服务（8001）未启动
  - 解决方法：执行 `python start_api.py`，再刷新页面重试
- 问题现象：决策质量下降/检索异常
  - 可能原因：向量库未初始化、模型调用失败
  - 解决方法：检查 `logs/decision_api.log`，确认知识库与模型配置可用
- 问题现象：页面显示数据与后端不一致
  - 可能原因：前端 `dataClient.js`、`decisionClient.js`、`conversationClient.js` 当前默认返回模拟数据
  - 解决方法：将客户端改为真实 HTTP 请求并联调接口

## 6. 安全与注意事项
- 决策 API 目前 CORS 为全开放（`allow_origins=["*"]`），生产环境必须限制为可信域名
- 本地日志写入 `logs/decision_api.log`，请按审计要求保留与轮转
- 输入数据单位要求为 `°C`（决策接口有单位校验）
- 前端“执行/忽略”按钮当前为示例逻辑，不会下发真实控制指令，生产接入前需补全权限与审计流程
- 对外部署前建议增加鉴权、限流、异常告警与 HTTPS

## 7. 附录
- 术语说明：
  - KPI：关键性能指标
  - RAG：检索增强生成
  - HyDE：假设性文档嵌入检索方法
  - Source Trace：决策依据溯源信息
- 联系方式：
  - 请按项目实际团队通讯录维护（当前仓库未统一维护联系人信息）
