# 系统使用手册 v3.0

## 1. 系统简介
- 系统名称：EnerGAI工业能源智能诊断平台
- 版本号：v3.0
- 适用对象：化工工艺工程师、值班操作员、运维与测试人员
- 主要功能概述：
  - 实时监控温度、压力、流量、能耗等关键工业指标
  - Transformer模型预测未来趋势，输出注意力权重热力图
  - RAG检索增强诊断，结合安全条款生成决策建议
  - 双引擎协同（预测+诊断），支持SSE实时推送
  - 智能助手会话能力（支持会话上下文）
  - **新增：实时趋势心电图 + 24小时历史趋势双视图**
  - **新增：AI智能助手独立页面，类ChatGPT交互体验**

## 2. 运行环境要求
- 操作系统：Windows 10/11、Linux、macOS（开发环境）
- 运行时：Python 3.10+（推荐）
- 浏览器/客户端：Chrome、Edge（推荐最新版本）
- 硬件要求：
  - 最低：4 核 CPU，8 GB 内存
  - 推荐：8 核 CPU，16 GB 内存（启用知识库检索和模型时）
- 网络要求：
  - 本地开发可离线运行前端模拟数据
  - 若启用外部模型服务，需可访问通义千问API接口

## 3. 安装与部署
### 3.1 前置准备
- 获取项目源码并进入项目根目录
- 准备 Python 环境并安装依赖（参见各模块requirements.txt）
- 确保端口可用：
  - 8000：后端 API 服务（`api/main.py`）
  - 3000：前端开发服务器（`frontend/`）

### 3.2 安装步骤

#### 后端安装
1. 打开终端进入项目根目录
2. 安装 Python 依赖：`pip install fastapi uvicorn pydantic sqlalchemy numpy`
3. 启动后端服务：`python api/main.py`

#### 前端安装 (v3.0 新增)
1. 进入前端目录：`cd frontend`
2. 安装 Node.js 依赖：`npm install`
3. 启动开发服务器：`npm run dev`
4. 浏览器访问：`http://localhost:3000/`

### 3.3 启动与停止
- 启动后端：
  - `python api/main.py`
  - 访问地址：`http://127.0.0.1:8000/`
- 启动前端：
  - `cd frontend && npm run dev`
  - 访问地址：`http://localhost:3000/`
- 停止命令：在对应终端按 `Ctrl + C` 停止服务

## 4. 功能使用说明
### 4.1 登录系统
- 当前版本未实现用户名/密码登录。
- 启动服务后，直接在浏览器访问首页即可进入系统主界面。

### 4.2 主界面说明
- 页面名称：EnerGAI工业能源智能诊断平台
- 主要区域：
  - 综合概览（KPI 卡片：温度、压力、余热回收、节能效果）
  - 趋势分析（**双视图**）：
    - 实时趋势：心电图式滚动显示，最多20个数据点
    - 过去24小时：历史数据折线图，每小时一个数据点
  - 能耗分析（柱状图：实际能耗 vs 节能目标）
  - 设备状态（健康度与状态）
  - 预警信息
  - 智能操作建议
  - **AI智能助手入口按钮**（点击跳转独立页面）
- 数据更新频率：
  - 时间每 1 秒刷新
  - 实时趋势每 3 秒自动滚动更新
  - 其他业务数据每 5 秒刷新一次

### 4.3 AI智能助手页面 (v3.0 新增)
- 入口：点击驾驶舱右下角"AI智能助手"按钮
- 页面布局（30% + 70% 左右分栏）：
  - 左侧：诊断报告卡片（设备名称、问题描述、原因分析、处置建议、知识来源）
  - 右侧：对话交互界面
    - 顶部标题"EnerGAI 智能助手”
    - 中间：对话记录区（AI气泡支持Markdown渲染和来源标注）
    - 底部：输入框 + 发送按钮
- 功能特性：
  - 新消息自动滚动到底部（类ChatGPT体验）
  - 仅显示AI头像，用户头像隐藏
  - 支持Markdown格式回复
  - 显示信息来源标注

### 4.4 核心功能操作
#### 4.4.1 监控与可视化
- 查看实时 KPI（温度、压力、流量、能耗）及预测值
- 查看趋势分析图
- 点击"导出CSV"下载当前数据

#### 4.4.2 智能决策建议
- 系统自动调用Transformer预测+RAG诊断
- 可执行操作：
  - "立即执行"：记录执行动作（当前为前端占位逻辑）
  - "忽略"：记录忽略动作（当前为前端占位逻辑）
  - "查看依据"：展示 `source_trace` 溯源信息
- 决策接口：`POST /api/v1/decision`

#### 4.4.3 智能助手会话
- 在"智能助手"输入问题后点击"发送"或按回车
- 系统返回助手回复并展示"操作依据（context_trace）"
- 会话接口：`POST /api/v1/conversation`
- 支持基于 `session_id` 的上下文继承

## 6. 常见问题与排查
- 问题现象：页面无法打开
  - 可能原因：后端服务未启动或端口冲突
  - 解决方法：重新执行 `python backend/main.py`，按控制台提示端口访问
- 问题现象：智能助手无响应或报错
  - 可能原因：决策服务（8006）未启动
  - 解决方法：执行 `python start_api.py`，再刷新页面重试
- 问题现象：决策质量下降/检索异常
  - 可能原因：向量库未初始化、模型调用失败
  - 解决方法：检查 `logs/decision_api.log`，确认知识库与模型配置可用
- 问题现象：页面显示数据与后端不一致
  - 可能原因：前端客户端默认返回模拟数据
  - 解决方法：将客户端改为真实 HTTP 请求并联调接口

## 7. 安全与注意事项
- 决策 API 目前 CORS 为全开放（`allow_origins=["*"]`），生产环境必须限制为可信域名
- 本地日志写入 `logs/decision_api.log`，请按审计要求保留与轮转
- 输入数据单位要求为 `°C`（决策接口有单位校验）
- 前端"执行/忽略"按钮当前为示例逻辑，不会下发真实控制指令，生产接入前需补全权限与审计流程
- 对外部署前建议增加鉴权、限流、异常告警与 HTTPS
